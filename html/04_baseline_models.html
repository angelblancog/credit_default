# Computamos las mtricas para ambos sets para ver si hay overfitting
sets = {
    "train": {"x": x_train, "y": y_train},
    "validation": {"x": x_val, "y": y_val}
}

rows = []

for model in models:

    # Entrenamiento
    name = model.__class__.__name__
    start_train = time.perf_counter()
    model.fit(x_train, y_train)
    end_train = time.perf_counter()
    
    eta_train = end_train - start_train
    
    write_model(model=model, name=name)
    
    print(f'Saved model "{name}"')
    
    # Clculo de mtricas contra ambos sets
    for set_name, set_data in sets.items():
        
        start_predict = time.perf_counter()
        y_pred = model.predict(set_data["x"])
        end_predict = time.perf_counter()

        eta_predict = end_predict - start_predict

        y_prob = model.predict_proba(set_data["x"])
        
        # Ccombinacin de diccionarios
        metrics = {
            "model": name,
            "set": set_name,
            "training_time (min)": eta_train/60,
            "predict_time (sec)": eta_predict} | get_metrics(y_true=set_data["y"], y_pred=y_pred, y_prob=y_prob)  # esto devuelve un diccionario

        # Pandas requiere una lista
        metrics =  {key: [value] for key, value  in metrics.items()}
        
        rows.append( pd.DataFrame(metrics) )

all_metrics = pd.concat(rows).round(4)

En problemas de clasificacin con conjuntos de datos desbalanceados, las mtricas de precisin y accuracy pueden ser engaosas porque pueden dar una imagen demasiado optimista del rendimiento del modelo. A mayores de estas, he considerado estas otras mtricas que aparecen el la tabla:

  • Time: el tiempo que tarda el modelo en conseguir objetivos es importante a la hora de contrastar si compensa su resultado o no. Puedo tener un modelo que haga predicciones genial pero que tarde demasiado tiempo en ejecutarse comparado con otro de resultados algo menores, lo que se traduce en ms costes para la organizacin.

  • Accuracy (Exactitud): calcula la fraccin de predicciones que un modelo realiz correctamente. Es una buena mtrica cuando tenemos un conjunto de datos balanceado, es decir, cuando el nmero de etiquetas de cada clase es similar (lo cual no es nuestro caso). Representa el porcentaje total de valores correctamente clasificados, tanto positivos como negativos.

  • Precission (Precisin): mide la proporcin de identificaciones positivas que fueron realmente correctas.

  • Recall (Sensibilidad): es bastante til cuando la clase de inters es la minoritaria. Mide la proporcin de positivos reales que se identifican correctamente.

  • F1 Score: media armnica de la precisin y la sensibilidad. Combina ambas mtricas en una sola cifra y es til cuando quieres buscar un equilibrio entre precisin y
    sensibilidad.

  • F2 Score: es una variante de la medida F que da ms peso a la sensibilidad que a la precisin.

  • ROC: la curva ROC es una representacin grfica que muestra el rendimiento de un modelo de clasificacin en todos los umbrales de clasificacin. Un AUC de 1 indica que el modelo tiene una capacidad perfecta para distinguir entre las clases. Un AUC de 0.5 indica que el modelo no tiene capacidad para distinguir entre las clases.

La eleccin de la mtrica depende del problema especfico que estemos tratando de resolver y de si consideramos ms importante en ese caso minimizar los falsos positivos o maximizar los verdaderos positivos. Veamos primero resultados y luego tratamos lo que he elegido hacer.

Se ha usado un Oversampling con SMOTE al 25% (por defecto viene al 50%).

En cuanto a las mtricas, en este caso se ha dado prioridad a minimizar los falsos negativos, puesto que le costaran mucho ms dinero a la organizacin y podran meterla en problemas de solvencia. Teniendo esto en cuenta, la mtrica que se ha priorizado es la f2_score, por ser la ms adecuada para esta premisa.

Comparando los grficos, se observa que los modelos tienen overfitting (se comportan muy bien en el train, pero en validacin su rendimiento es mucho menor). Esto ocurre de manera ms pronunciada en los modelos ms complejos porque, al entrenar, se sobreajustan al dataset de train, no pudiendo generalizar bien a nuevos datos. Tambin es posible que se de por falta de entrenamiento o un entrenamiento anmalo, por exceso de ruido en los datos de muestra, o porque hayan entrenado con muchos datos.

Para evitar el overfitting, suelen utilizarse tcnicas como la Data Augmentation o la validacin cruzada.

El modelo que mejor se comporta teniendo en cuenta tambin el tiempo es la LogisticRegression (el GradientBoostingClassifier tarda significativamente ms pese a obtener resultados similares).
